# bridge_updater

## О проекте

`bridge_updater` — это утилита на Go для автоматического обновления и записи мостов Tor. Она загружает данные из двух источников:

- `direct` — прямые URL-адреса, скачиваемые с помощью `curl`
- `repos` — локальные git-репозитории, из которых обновляются файлы мостов с помощью `git pull`

Далее содержимое файлов преобразуется и записывается в выходные файлы в формате, ожидаемом Tor.

Также: частично совместим с Zapret и практически полностью совместим с  VLESS-клиентами. VLESS-клиент рекомендуется перевести из TUN/VPN-режима в режим прокси (так tor-proxy bridge_updater'а будет работать быстрее). Про Zapret: см. подробности ниже.

## ДИСКЛЕЙМЕР

- Это pet-проект для демонстрации навыков работы автора с `golang` (включая использование goroutines), `git`, `bash` и `vagrant`.
- Автор не несет ответственности за использование этой утилиты в злонамеренных целях или в целях обхода блокировок и не одобряет использования утилиты подобным образом.
- Виртуальная среда Vagrant прилагается исключительно для удобства тестирования отдельных компонентов `bridge_updater`, так как он разрабатывался для использования нативно на Linux.
- Yggdrasil-мосты используются для обеспечения стабильного соединения с Tor в целях тестирования и демонстрации корректной работы через прокси функции `checkInternetReachabilityOneshot` из модуля `actions/checker.go`, а также функций из модуля `actions/fetcher.go`.
- Примеры использования программы для доступа к популярным ресурсам приведены для удобства тестирования и демонстрации корректности работы Tor, и, как следствие, для демонстрации корректности работы `actions/writer.go` с конфигурацией Tor.
- Пример использования программы для обеспечения доступа Tor Browser к сети Tor приведен для демонстрации корректной обработки получаемых программой мостов.

## Использование

Практически все приложения, которые поддерживают пользовательские настройки прокси, могут использовать `SOCKS5`-прокси с адресом `127.0.0.1` и портом `6969`. Все перечисленные ниже примеры релевантны для использования с предоставленной виртуальной средой Vagrant.

### Примеры

- Telegram: да. Однако звонки не будут работать. Чтобы настроить прокси: Настройки -> Продвинутые -> Тип соединения -> Добавить прокси -> тип `SOCKS5`, хост `127.0.0.1`, порт `6969`.
- Discord: нет. Иcпользуйте другие инструменты для получения доступа (к примеру, [zapret-discord-youtube](https://github.com/Flowseal/zapret-discord-youtube)).
- Keybase: да. Добавьте прокси в продвинутых настройкак, как в Telegram.
- Обход "белых списков": нет. Используйте VLESS-клиент со специализированными конфигами.
- Браузер: да. Установите расширение FoxyProxy и добавьте `SOCKS5` прокси, хост `127.0.0.1`, порт `6969`. Не используйте Яндекс Браузер.
  - ChatGPT: да.
  - Google Gemini: да.
  - Anthropic: да.
  - TikTok: да.
  - Youtube: да. Важно: если вы используете [zapret-discord-youtube](https://github.com/Flowseal/zapret-discord-youtube), тогда отключите tor-прокси для браузера, в котором вы собираетесь смотреть Youtube, или выключите zapret, или используйте 2 разных браузера: один — для Youtube (без прокси), второй — для других ресурсов (с прокси).
  - Tor Browser: да. Просто войдите в терминал виртуальной машины через `vagrant ssh` или `enter.bat`, введите команду `bridges`, скопируйте и вставьте несколько мостов в соответствующую секцию в Tor Browser в настройках подключения (также см. Руководство по менеджменту Vagrant-машиной ниже). Не используйте `127.0.0.1:6969` как прокси в Tor Browser!
  - .onion-ресурсы: возможно, но крайне не рекомендуется. Используйте Tor Browser.
  - Spotify, Soundcloud: возможно, но неудобно. Лучше используйте VLESS-клиент в proxy-режиме.
  - Любой российский сервис (Ozon, Wildberries, Yandex,..): нет.

## Виртуализация

К проекту для удобства развертывания и работы прилагается виртуальная среда на базе Vagrant, которая автоматически разворачивает рабочую Ubuntu-машину с установленным Tor-прокси и всем необходимым для программы.

### Дополнительные детали

- Поддержка Tor-прокси IPv6 включена по умолчанию. При необходимости вы можете выключить ее, закомментировав нужные строки в vagrant/torrc.
- Если нужно вручную отредактировать конфигурацию для `bridge_updater` перед запуском машины, то она находится в: `vagrant/brupd_conf.json`. По умолчанию там настроено несколько источников для obfs4 мостов.
- На данный момент для Vagrant-машины не поддерживаются webtunnel мосты.
- Не рекомендуется использовать vanilla мосты.
- В худшем случае, если доступа ко всем полученным мостам нет, также предусмотрена поддержка мостов в Yggdrasil network. Вы можете отключить их с помощью команды: `ygg off`
- Мосты обновляются каждый час по умолчанию.
- Tor-прокси будет доступен с хоста по адресу: `127.0.0.1:6969` (SOCKS5).
- Если у вас Windows, то скрипт `win_install_deps.ps1` **ВЫКЛЮЧИТ** Hyper-V и Windows Sandbox во избежание конфликтов с Virtualbox.

## Quick Start (через Vagrant)

### Windows

1. Установите вручную Vagrant и Virtualbox или запустите `install.bat` в корневой папке проекта с правами администратора.

## Команды

Утилита поддерживает следующие команды:

- `update` — полное обновление и запись мостов
- `fetch` — только загрузка данных (`git pull` и/или `curl`)
- `write` — только запись уже полученных файлов в выходные файлы
- `check` — проверка конфигурации, обязательных инструментов и подключения к интернету

### Флаги

- `-c, --config` — путь к файлу конфигурации
- `-l, --loglevel` — уровень логов: `debug`, `info`, `warn`, `error` (по умолчанию `info`)
- `-r, --repos-only` — выполнять только секцию `repos`
- `-d, --direct-only` — выполнять только секцию `direct`
- `-i, --ignore-internet-reachability-tests` — пропускать тесты доступности интернета
- `-t, --no-tor-reload` — не перезагружать Tor после команды `write` или `update`.

> Флаги `--repos-only` и `--direct-only` нельзя использовать одновременно. Также они будут автоматически проигнорированы при запуске команды `check`.
> Флаг `--no-tor-reload` будет проигнорирован при запуске `check` и `fetch`.

## Конфигурация вручную

Файл конфигурации должен соответствовать JSON-схеме, встроенной в проект. Пример конфигурации содержится в `config/conf_example.json`.

### `direct`

В разделе `direct` указываются URL и путь для сохранения исходного файла, а также список файлов вывода.

```json
"direct": {
  "https://example.com/bridge.txt": {
    "dest": "/tmp/bridge.txt",
    "outputs": ["/etc/tor/bridge.conf"]
  }
}
```

Каждый объект `direct` содержит:

- `dest` — локальный путь для загрузки исходного файла
- `outputs` — массив файлов, в которые будет записан результат

### `repos`

В разделе `repos` указывается путь к локальному git-репозиторию и набор файлов внутри репозитория с соответствующими выходными файлами.

```json
"repos": {
  "/path/to/repo": {
    "relative/path/to/source.txt": ["/etc/tor/bridge1.conf", "/etc/tor/bridge2.conf"]
  }
}
```

Правила для `repos`:

- путь к репозиторию не должен оканчиваться слэшем `/`
- путь к файлу внутри репозитория должен быть относительным
- репозиторий должен быть валидным git-репозиторием

Также:

- Файлы в конфигурации (как выходные, так и для загрузки) не должны дублироваться

### Формат файлов с мостами

Пример корректного входного файла с мостами:

```properties
#comment line
#comment line

obfs4 example.com:443 0123456789ABCDEF0123456789ABCDEF01234567 cert=ABCDEFG iat-mode=0
webtunnel example.com:443 0123456789ABCDEF0123456789ABCDEF01234567

```

## Руководство по менеджменту Vagrant-машиной

Предполагается, что основная работа с программой будет вестись в виртуальной среде Vagrant.

### Управление машиной на Linux

Используйте стандартные команды Vagrant в терминале:

- `vagrant up` — запустить виртуальную машину
- `vagrant halt` — остановить виртуальную машину
- `vagrant destroy` — удалить виртуальную машину
- `vagrant status` — проверить статус машины
- `vagrant ssh` — подключиться к машине по SSH

### Управление машиной на Windows

Для удобства управления на Windows предусмотрены .bat-скрипты:

- `up.bat` — запустить виртуальную машину
- `halt.bat` — остановить виртуальную машину
- `destroy.bat` — удалить виртуальную машину
- `enter.bat` — подключиться к машине по SSH
- `manage.bat` — открыть интерактивное меню управления (поддержка английского и русского языков)
- `service.bat` — управление автозапуском машины (создать задачу запуска при входе в систему)

### Команды внутри виртуальной машины

Внутри машины автоматически настроены полезные алиасы и функции для работы с программой:

#### Основной алиас: `brupd`

Запускает программу `bridge_updater` с предустановленной конфигурацией и уровнем логирования:

```bash
brupd check
brupd fetch
brupd write
brupd update
brupd update --repos-only
brupd update --direct-only
brupd update --ignore-internet-reachability-tests
```

#### Функция: `brupd-tor`

Запускает программу через Tor-прокси (все используемые программой сетевые утилиты идут через Tor-прокси):

```bash
brupd-tor update
brupd-tor fetch -d
```

#### Справка: `brupd-info`

Выводит справку на русском или английском языке и список полезных команд:

```bash
brupd-info ru
brupd-info en
```

#### Функция: `reconfigure-daemon`

Позволяет переключать службу `brupd` между режимами работы через Tor и напрямую. Принимает аргумент `tor` или `std`:

```bash
reconfigure-daemon tor
reconfigure-daemon std
```

#### Функция: `print-service-mode`

Показывает текущий режим работы службы `brupd` (через Tor или напрямую):

```bash
print-service-mode ru
print-service-mode en
```

#### Функция: `ygg`

Позволяет включать, отключать поддержку мостов в сети yggdrasil, а также переключаться на использование мостов только в yggdrasil. Принимает аргумент `on`, `off` или `only`:

```bash
ygg on
ygg off
ygg only
```

#### Функция `journal`

Позволяет просматривать логи службы Tor. Первый аргумент может быть `bootstrap` (просмотр только bootstrap-логов) или `all` (просмотр всех логов). Также можно использовать дополнительный флаг `-i` для просмотра логов в интерактивном режиме:

```bash
journal bootstrap
journal all
journal bootstrap -i
journal all -i
```

#### Функция `brupd-tor-onfailure`

Позволяет контролировать поведение `brupd-tor.service`. Принимает в качестве аргумента `brupd`, `none` или `status`. `brupd` позволит попытаться запустить `brupd.service`, если `brupd-tor.service` потерпит неудачу и попытаться получить мосты, не используя tor (поведение по умолчанию). `none` отключит такое поведение. `status` напечатает, как `brupd-tor.service` ведет себя сейчас.

```bash
brupd-tor-onfailure brupd
brupd-tor-onfailure none
brupd-tor-onfailure status
```

#### Дополнительные команды

- `ipcheck` — проверить текущий IP-адрес выходного узла Tor
- `status` — проверить статус Tor-сервиса
- `restart` — перезагрузить Tor-сервис
- `bridges` — показать все текущие настроенные мосты
- `sudo nyx` — запустить сетевой монитор Tor
